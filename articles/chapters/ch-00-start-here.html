Start Here

Never written a line of code? This is your on-ramp. Before diving into C++ syntax, this chapter builds the mental model every programmer needs — how computers actually work, what the terminal is, and why any of this makes sense.

← Back to Roadmap

Who this chapter is for

If you have never programmed before — not even a "Hello, World" — start here. Everything on this site assumes you can read a terminal, understand what a file path is, and have a rough idea of what happens when you run a program. This chapter fills those gaps before the real C++ work begins.

If you've coded before in any language, even a little, you can skip straight to Chapter 1.

How to read these notes

Throughout the articles you'll see color-coded callout blocks. Each type means something specific:

format_quote Quote

"The input stream reads characters until it encounters whitespace — the rest stays buffered for the next read." — Roberts, Ch. 4

code Exercise

Write a function that accepts a string reference and reverses it in place without using any standard library algorithms.

lightbulb Tip

Passing by const reference is almost always faster for large objects since it avoids copying data entirely.

warning Danger

Reading from an uninitialized variable is undefined behavior — the compiler is allowed to do anything, including appearing to work correctly.

The most important thing to read first

MIT's "Missing Semester" course covers everything a CS degree assumes you already know but never actually teaches: the shell, text editors, version control, debugging, and how to think about a computer as a tool. It is free, short, and genuinely one of the best things you can do before writing your first C++ program.
