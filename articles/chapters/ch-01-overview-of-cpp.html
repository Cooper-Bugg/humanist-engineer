Overview of C++

Notes and checkpoints for getting a C++ program from source code to an executable: compilation, basic types, expressions, and the shape of a small program.

← Back to Roadmap
lightbulb Tip

These notes use color-coded callout blocks throughout: yellow for quotes, blue for exercises, green for tips, and orange for gotchas. Hover the ? in the bottom-right corner any time for a quick reference. First time here? Ch. 0 has a full walkthrough.

What I’m trying to learn here

  • What the compiler and linker do (at a high level)
  • How a C++ program is structured (headers, translation units, main())
  • Core types + literals + expressions
  • Common build/run workflow and the mental model for “edit → compile → run”

Key terms

  • translation unit: the source file after the preprocessor runs
  • compiler vs linker: compile each unit, then link into one program
  • undefined behavior: the standard doesn’t define what happens (avoid it)
